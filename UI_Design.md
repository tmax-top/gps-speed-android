# GPS Speed UI 设计与交互文稿 (V1)

## 1. 全局设计规范 (Design System)

骑行场景在户外，光线复杂，因此 UI 必须具备**高对比度、大字体、防眩光**的特性。

### 1.1 色彩规范 (Color Palette)
- **背景色 (Background)**: `#121212` (深邃黑，省电且在户外阳光下对比度高)
- **卡片/悬浮层 (Surface)**: `#1E1E1E` (深灰)
- **主强调色 (Primary)**: `#00E676` (荧光绿，用于正常骑行状态、开始按钮、GPS信号良好)
- **警告色 (Warning/Error)**: `#FF5252` (亮红色，用于停止按钮、GPS信号弱/丢失提示)
- **主文本色 (Text Primary)**: `#FFFFFF` (纯白，用于速度数字、主要标题)
- **次文本色 (Text Secondary)**: `#9E9E9E` (浅灰，用于单位、说明文字、未激活图标)

### 1.2 字体规范 (Typography)
- **数字字体**: 强烈建议使用 **DIN Alternate** 或 **Roboto Mono** (等宽数字字体，防止速度跳动时整个文本框宽度抖动)。
- **常规中文字体**: 系统默认无衬线体 (San Francisco / Roboto / 苹方)。
- **字号阶梯**:
  - `Display` (主速度数字): `120sp` (极粗体 Bold)
  - `Title` (页面标题/数据面板数值): `24sp` (中黑 Medium)
  - `Body` (常规说明): `16sp` (常规 Regular)
  - `Caption` (单位 km/h、辅助说明): `14sp` (常规 Regular)

## 2. 页面线框图与布局结构

### 2.1 主测速页 (Main Speedometer)

这是用户打开 App 后的默认页面。

```text
+---------------------------------------------------+
|  [设置图标]                      [悬浮窗] [全屏]  |  <-- 顶部操作栏 (24dp icon)
|                                                   |
|       [GPS信号图标] 定位良好 (绿色)               |  <-- 状态提示 (14sp)
|                                                   |
|                                                   |
|               0                                   |  <-- 速度大数字 (120sp, 纯白)
|                                                   |
|              km/h                                 |  <-- 单位 (16sp, 浅灰)
|                                                   |
|---------------------------------------------------|
|  最高时速       |  骑行时长      |  当前里程      |  <-- 数据面板 (卡片背景 #1E1E1E)
|  0.0            |  00:00:00      |  0.0           |  <-- 数据值 (24sp, 纯白)
|---------------------------------------------------|
|                                                   |
|                                                   |
|           (  开始骑行  )                          |  <-- 主操作按钮 (高度 64dp, 圆角 32dp, 荧光绿背景)
|                                                   |
|                                                   |
|           [历史记录入口]                          |  <-- 底部次要入口 (16sp, 带下划线)
+---------------------------------------------------+
```
**交互说明：**
- 点击 **[开始骑行]** 后，按钮变为左右两个半宽按钮：**[暂停]** (灰色) 和 **[结束并保存]** (红色)。
- 点击 **[全屏]** 图标，页面无缝转场至全屏模式。
- 点击 **[悬浮窗]** 图标，App 退到后台，并在桌面弹出悬浮窗。

### 2.2 全屏测速页 (Fullscreen Mode)

横竖屏自适应，去除所有干扰信息，纯粹做仪表盘。

```text
+---------------------------------------------------+
| [退出全屏]                             [锁布局]   | <-- 顶部半透明悬浮图标 (3秒后自动隐藏)
|                                                   |
|                                                   |
|                                                   |
|                      0                            | <-- 极致大数字 (占据屏幕宽度的 80%)
|                                                   |
|                                                   |
|                                                   |
|                     km/h                          | <-- 单位
|                                                   |
|  [GPS: 良好]                        [最高: 45.2]  | <-- 底部左右角落微小提示 (12sp, 浅灰)
+---------------------------------------------------+
```
**交互说明：**
- 屏幕保持 **常亮 (Keep Screen On)**。
- 顶部图标在无操作 3 秒后自动淡出隐藏，点击屏幕任意位置唤醒。
- 点击 **[锁布局]**，屏幕锁定，防误触（点击屏幕提示“长按解锁”）。

### 2.3 悬浮窗 (Floating Window)

小巧，可拖拽，点击可穿透或展开。

```text
+--------------------+
|   25               | <-- 速度 (32sp)
|  km/h              | <-- 单位 (10sp)
+--------------------+
```
- **形态**: 药丸形 (Pill-shaped) 或正圆形。
- **背景**: `#000000`，透明度 `70%`，带微弱的白色描边边框。
- **交互**: 
  - 按住任意位置可全屏自由拖动。
  - 单击悬浮窗 -> 返回 App 主界面。
  - 长按悬浮窗 -> 弹出小菜单：`[关闭]` `[返回App]`。

简洁的时间轴风格列表。

```text
+---------------------------------------------------+

|  今天 08:30                                       |
|  时长: 45分钟   |   里程: 12.5 km                 |
```

## 3. 动效与触觉反馈 (Motion & Haptics)

1. **速度跳动动画**:
   - 当速度从 20 变为 25 时，数字不要硬切，应使用属性动画（ValueAnimator）在 `300ms` 内平滑滚动增加 (20 -> 22 -> 24 -> 25)。
2. **震动反馈 (Haptics)**:
   - 点击“开始骑行”和“结束骑行”时，触发系统震动（`HapticFeedbackConstants.LONG_PRESS`），给骑行戴手套的用户以明确的物理反馈。
3. **状态切换警告**:
   - 当 GPS 信号丢失时，速度数字变为浅灰色或显示为 `--`，且顶部信号提示闪烁红光。

## 4. 适配说明
- **深色模式**: App 强制全局深色模式（Force Dark Theme），无需适配浅色，符合产品场景需求。
- **屏幕适配**: 测速大数字需使用 `autoSizeTextType`，确保在小屏手机上不会被截断，在大屏手机上能充分撑满。
